# 个人博客网站设计方案

## 一、设计理念

**核心原则：少即是多（Less is More）**

- 极简主义美学，去除一切不必要的装饰
- 内容为王，让文字、图片、视频成为主角
- 流畅的交互体验，注重细节动效
- 响应式设计，完美适配各种设备

## 二、视觉设计

### 2.1 色彩方案

**浅色模式（默认）**
- 主色：`#1a1a1a` (深灰黑，用于标题和重要文字)
- 次要色：`#666666` (中灰，用于正文)
- 辅助色：`#999999` (浅灰，用于次要信息)
- 背景色：`#ffffff` (纯白)
- 强调色：`#0066cc` (蓝色，用于链接和交互元素)
- 分割线：`#e5e5e5` (极浅灰)

**深色模式**
- 主色：`#e5e5e5` (浅灰白)
- 次要色：`#a0a0a0` (中灰)
- 辅助色：`#707070` (深灰)
- 背景色：`#0a0a0a` (近黑)
- 强调色：`#4d9fff` (亮蓝)
- 分割线：`#2a2a2a` (深灰)

### 2.2 字体系统

**中文字体栈**
```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB",
             "Microsoft YaHei", "WenQuanYi Micro Hei", sans-serif;
```

**英文/代码字体栈**
```css
font-family: "SF Mono", "Fira Code", "Consolas", "Monaco", monospace;
```

**字号层级**
- 标题 H1: 2.5rem (40px)
- 标题 H2: 2rem (32px)
- 标题 H3: 1.5rem (24px)
- 正文: 1.125rem (18px)
- 小字: 0.875rem (14px)

**行高**
- 标题: 1.3
- 正文: 1.8
- 代码: 1.6

### 2.3 间距系统

采用 8px 基准网格系统：
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- 2xl: 48px
- 3xl: 64px
- 4xl: 96px

## 四、交互设计

### 4.1 导航栏

**桌面端**
- 固定在顶部，背景半透明毛玻璃效果（backdrop-filter: blur(10px)）
- 左侧：Logo/名字
- 右侧：导航链接（文章、摄影、视频、关于）+ 主题切换按钮
- 滚动时自动隐藏，向上滚动时显示
- 当前页面链接有下划线或高亮

**移动端**
- 汉堡菜单按钮（右上角）
- 点击展开全屏菜单，从右侧滑入
- 菜单项垂直排列，大字号，易于点击

### 4.2 主题切换

**切换方式**
- 导航栏右侧有太阳/月亮图标按钮
- 点击时有平滑的颜色过渡动画（transition: all 0.3s ease）
- 记住用户选择（localStorage）

**动画效果**
- 图标旋转 180 度
- 背景色、文字色渐变过渡
- 避免闪烁

### 4.3 页面过渡

**路由切换动画**
- 淡入淡出（fade in/out）
- 持续时间 300ms
- 使用 Framer Motion 或 CSS transitions

### 4.4 滚动体验

**平滑滚动**
```css
html {
  scroll-behavior: smooth;
}
```

**滚动动画**
- 元素进入视口时渐入（Intersection Observer API）
- 图片懒加载
- 无限滚动或分页（文章列表）

### 4.5 微交互

**按钮**
- Hover: 背景色变深，轻微放大（scale: 1.02）
- Active: 轻微缩小（scale: 0.98）
- 过渡时间 200ms

**卡片**
- Hover: 上浮 4px，阴影加深
- 过渡时间 300ms

**链接**
- Hover: 下划线从左到右展开动画
- 颜色变化

**图片**
- 加载时显示骨架屏或模糊占位符
- 加载完成后渐入

### 4.6 反馈机制

**加载状态**
- 页面切换时顶部显示进度条
- 数据加载时显示 Spinner

**错误提示**
- Toast 通知（右上角）
- 自动消失（3-5秒）

**成功提示**
- 评论提交成功、复制成功等
- 简短的 Toast 通知

## 五、响应式设计

### 5.1 断点设置

```css
/* 移动端 */
@media (max-width: 640px) { }

/* 平板 */
@media (min-width: 641px) and (max-width: 1024px) { }

/* 桌面端 */
@media (min-width: 1025px) { }
```

### 5.2 布局适配

**移动端（< 640px）**
- 单列布局
- 导航改为汉堡菜单
- 字号适当缩小（H1: 2rem, 正文: 1rem）
- 图片瀑布流改为 1 列
- 卡片间距减小（16px）

**平板（641px - 1024px）**
- 双列布局（部分区域）
- 导航保持横向
- 图片瀑布流 2 列

**桌面端（> 1024px）**
- 最大内容宽度 1200px，居中
- 图片瀑布流 4 列
- 文章详情页可显示侧边栏目录

### 5.3 触摸优化

- 按钮最小点击区域 44x44px
- 滑动手势支持（图片轮播、菜单）
- 避免 hover 效果在移动端的问题

## 六、性能优化

### 6.1 图片优化

- 使用 Next.js Image 组件自动优化
- WebP 格式优先，降级到 JPEG/PNG
- 响应式图片（srcset）
- 懒加载（loading="lazy"）
- 模糊占位符（blurDataURL）

### 6.2 代码分割

- 路由级别代码分割（Next.js 自动）
- 动态导入重组件（React.lazy + Suspense）
- 第三方库按需加载

### 6.3 缓存策略

- 静态资源长期缓存
- API 数据使用 SWR 或 React Query
- Service Worker（可选）

### 6.4 首屏加载

- 关键 CSS 内联
- 字体预加载
- 骨架屏
- SSR/SSG（Next.js）

## 十、总结

这个设计方案的核心特点：

1. **极简美学**：去除冗余，突出内容
2. **舒适阅读**：合理的字号、行高、宽度
3. **流畅交互**：细腻的动画和反馈
4. **响应式**：完美适配各种设备
5. **高性能**：快速加载，流畅体验
6. **可扩展**：清晰的组件结构，易于维护

建议先实现基础框架和文章功能，再逐步添加摄影、视频等模块。每个阶段都可以根据实际需求调整。
